# 「下一场发布会 / 发布时刻」· 一年商业规划（以盈利为目标）

> 制定日期：2026-06-11　|　周期：2026-06 ～ 2027-05
> 定位：科技发布会日历工具（网站 + 微信小程序），中英双语，主体杭州有限公司
> 核心判断：**工具本身变现弱，必须用「内容获客 → 工具留存 → 多渠道变现」的飞轮**，单靠日历 App 难盈利。

---

## 一、商业模式：变现路径（按"对本产品的契合度+落地难度"排序）

| 优先级 | 渠道 | 说明 | 起量门槛 |
|---|---|---|---|
| ⭐⭐⭐ | **CPS 导购带货** | 发布会后，新机/数码产品给京东联盟/淘宝客/拼多多带货链接，按成交抽佣。**与"发布会"场景天然契合**——用户看完发布会正想买 | 低，有少量流量即可 |
| ⭐⭐⭐ | **小程序流量主 + 网站广告** | 微信小程序"流量主"(banner/插屏/激励视频)、网站联盟广告。按曝光/点击计费 | 低（小程序需累计1000+用户开通流量主）|
| ⭐⭐ | **内容商单 / 品牌赞助** | 给数码品牌做"发布会前瞻/速报"软广，或卖发布会"赞助高亮位" | 中，需要内容影响力 |
| ⭐⭐ | **会员订阅** | 高级版：全量提前提醒、独家整理、去广告、日历导出、API | 中，需验证付费意愿 |
| ⭐ | **B 端数据/API** | 把发布会日历数据做成 API 卖给科技媒体/MCN/导购站 | 高，后期 |

**结论**：Q1-Q2 主攻 **CPS + 流量主广告**（启动快、与场景契合）；Q3 起叠加 **会员订阅 + 内容商单**；B 端 API 作为 Year 2 方向。

---

## 二、增长引擎：内容矩阵（这是关键，工具不会自己传播，内容才会）

围绕每一场大发布会做"三段式"内容，全平台分发，把流量导回小程序/网站：

```
发布会前：前瞻（看点、时间、怎么看直播）→ 引导"设提醒"
发布会中：速报（重点发布、价格）       → 实时拉访问
发布会后：总结 + 购买导购（CPS链接）   → 变现 + 沉淀用户
```

**分发渠道**（按 ROI 排序，集中做 2-3 个，别铺太开）：
1. **微信公众号**（沉淀私域、可接商单、导流小程序）— 必做
2. **小红书 / 视频号**（科技数码内容流量大、易起号）— 主攻其一
3. 抖音/B站（视频精力够再做）

**内容节奏**：贴着发布会日历排期（秋季 9-11 月是高峰：Apple、华为、小米、Meta Connect、双11），淡季做盘点/横评/榜单。

## 三、季度里程碑（Quarter-by-Quarter）

### Q1（2026-06 ～ 08）｜基建 + 上线 + 内容起号
- ✅ 公司注册、对公户、税务报到、代账到位
- ✅ **ICP 企业备案** → fabushike.com HTTPS 正式上线（解除目前"未备案不能公网访问"）
- ✅ **小程序企业主体上线**（解决名称/类目限制）
- ✅ 接入 **CPS 联盟**（京东联盟/淘宝客）+ 申请小程序**流量主**
- ✅ 开 **公众号 + 1个内容平台**，绑定夏季发布会(WWDC后、ChinaJoy、Gamescom)做内容
- **KPI**：双端上线；种子用户 500-1000；内容账号 0→1；收入≈0（投入期）

### Q2（2026-09 ～ 11）｜秋季发布会高峰 = 流量+变现引爆季
- 🔥 借 **Apple 9月、华为/小米、Meta Connect、双11** 密集发布会大量产内容冲流量
- 🔥 **CPS 带货第一波**（秋季新机 + 双11 购物季）→ 第一笔实质收入
- 上线**会员订阅**（小步测试付费意愿）
- **KPI**：MAU 5k-2万；CPS+广告 ¥2k-1万/月；公众号粉丝 5k-1万

### Q3（2026-12 ～ 2027-02）｜留存深化 + 年度内容 IP
- 出 **「2026 科技发布会年度盘点 + 2027 前瞻」**（爆款内容/引流钩子，可做付费报告）
- 借 **CES / MWC** 继续产内容
- 优化会员权益、广告填充、首批商单
- **KPI**：付费会员首批 100-500；**月收入打平硬成本**

### Q4（2027-03 ～ 05）｜规模化 + 盈利验证
- **品牌商单/赞助位**落地（1-3 单）；B 端 API 试水
- 全年复盘，验证≥2 条可持续变现渠道
- **KPI**：月收入 ¥1-3万；**月度现金流打平或小幅盈利**

## 六、关键指标（每月盯）

- **流量**：网站 UV、小程序 MAU/DAU、内容账号粉丝/阅读
- **留存**：次周留存、订阅提醒人数（产品核心价值锚点）
- **变现**：CPS 成交额/佣金、广告收入、会员付费数、ARPU
- **内容**：每场发布会内容产出数、单篇最高阅读、导流转化率

## 七、风险与对策

| 风险 | 对策 |
|---|---|
| **内容起不来 = 全盘没流量** | 集中精力做 1 个内容平台做透，别铺开；蹭每场大发布会热点；标题/选题对标头部数码号 |
| 流量低则广告/CPS 都微薄 | 先不靠广告养活，把 CPS（高客单数码）做成主力；流量未起前别投流烧钱 |
| 个人精力有限（产品+内容+运营） | 产品已基本成型，重心转内容运营；数据更新已半自动；考虑找 1 个兼职内容搭子 |
| 合规：UGC评论、CPS导购资质 | 评论已有敏感词+微信内容安全；CPS 用正规联盟；经营性变现前补 ICP 经营许可（增值电信）|
| 季节性强（秋季爆、淡季冷） | 淡季做盘点/横评/榜单常青内容，平滑流量 |
