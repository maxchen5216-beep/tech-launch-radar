> 前提（用户已确认）：图文+视频都能产出；月获客预算 ≤¥1000（以免费内容为主、小额加热为辅）；solo 运营；国内为主、企业主体。

## Context
产品已就绪（网站 fabushike.com + 微信小程序：发布会日历/提醒/评论/悄悄新品/往期总结）。成败不在技术在流量。本规划解决"用户从哪来"，与 `BUSINESS-PLAN-Y1.md` 的变现飞轮咬合：**内容获客 → 工具留存（设提醒）→ 自动召回 → 复访带货/看总结 → 分享拉新**。

## 一、目标人群（按变现价值排序）
1. **想买新机的"等等党"**（最值钱，CPS 直接契合）——关心手机/耳机/影像新品何时发、值不值得等
2. **数码爱好者/极客**——追前沿、AI、机器人、折叠屏
3. **科技从业者/开发者**——WWDC/IO/GTC/Build 等
4. **自媒体/内容创作者**——需要"发布会选题日历"（B 端潜力）
5. **行业研究/投资**——关注产业发布节奏
> 选题与获客优先向 #1/#2 倾斜：获客的同时养 CPS 变现盘。

## 二、增长飞轮（工具型产品的核心逻辑）
```
蹭发布会热点产内容 → 引导"设提醒"(留存钩子) → 发布会临近自动提醒召回
→ 用户复访看速报/总结 + 点带货链接(变现) → 分享日历给同好(拉新) → 下一场再循环
```
关键：**每一场科技发布会都是一次天然的"拉新+召回+变现"机会**，节奏自带。
